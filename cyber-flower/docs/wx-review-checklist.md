# 📋 微信小程序审核准备清单

> 赛博养花 提交微信审核前的自查项

## 类目选择

**建议类目**: 工具 > 信息查询 / 生活服务 > 休闲娱乐

> ⚠️ 如果涉及UGC(用户对话/留言)，需要「社交」类目，需企业资质

## 功能自查

- [ ] 无政治敏感内容
- [ ] 无诱导分享 (如"分享到3个群才能领种子" ❌)
- [ ] 无虚拟支付违规 (iOS虚拟支付限制)
- [ ] 用户生成内容有审核机制
- [ ] 有隐私政策页面
- [ ] 有用户协议页面
- [ ] 有投诉/举报入口
- [ ] 获取用户信息时有明确授权

## 技术合规

- [ ] 服务器域名已配置 (request/uploadFile/downloadFile)
- [ ] 所有网络请求使用HTTPS
- [ ] 未使用废弃API
- [ ] 小程序包体积 < 2MB (主包) + 20MB (总)
- [ ] 基础库版本设置合理 (≥2.0)
- [ ] 无WebView加载http页面
- [ ] 未使用web-view加载第三方页面

## 内容安全

- [ ] AI生成内容有审核过滤
- [ ] 图片上传有内容审核
- [ ] 用户输入有敏感词过滤
- [ ] 无自动关注/自动点赞行为
- [ ] 无虚假信息/虚假数据

## 隐私合规

- [ ] 有完整的隐私保护指引
- [ ] 用户数据可删除
- [ ] 用户数据可导出
- [ ] 收集用户信息时明确告知用途
- [ ] 符合《个人信息保护法》

## 提交材料

- [ ] 小程序截图 (至少5张，含主要功能界面)
- [ ] 功能演示视频 (可选但推荐)
- [ ] 测试账号 (如需要登录)
- [ ] 类目资质证明 (如需要)
- [ ] 简介/描述文案

## 审核可能被拒的点

1. **服务类目不符** — 选择正确类目
2. **功能不完整** — Demo感太强会被拒
3. **UI太简陋** — 至少要美观、可用
4. **隐私政策缺失** — 必须有隐私政策页面
5. **诱导行为** — 不能强制分享/关注
6. **内容安全** — 无审核机制会被拒
7. **支付违规** — iOS端虚拟支付问题
8. **代码质量问题** — 频繁crash会被拒

---

> 建议: 首次提交前进行内部3-5人测试，确保核心流程完整可用
