## 2. Phase 0: 项目初始化 (Week 1-2)

### Task 0.1: 微信小程序项目脚手架

- **目标**: 搭建微信小程序原生+TypeScript开发环境
- **涉及文件**: `mini-program/` (全部初始化文件)
- **实现步骤**:
  1. 注册微信小程序账号，获取AppID
  2. 初始化原生TS项目结构
  3. 配置 `project.config.json` / `tsconfig.json`
  4. 安装MobX + miniprogram-computed
  5. 配置ESLint + Prettier
  6. 搭建基础目录结构
- **测试要求**: `npm run build` 成功，开发工具可预览
- **完成标准**: 项目脚手架可运行，基础页面可跳转

### Task 0.2: 后端项目脚手架

- **目标**: 搭建NestJS后端项目 + 微信云开发环境
- **涉及文件**: `server/` (全部初始化文件)
- **实现步骤**:
  1. 初始化NestJS monorepo项目
  2. 配置TypeScript严格模式
  3. 配置微信云开发环境
  4. 搭建Docker Compose本地开发环境 (MongoDB + Redis)
  5. 配置日志系统 (Winston)
  6. 配置环境变量管理
- **测试要求**: `npm run start:dev` 成功，`/health` 端点返回200
- **完成标准**: 后端可启动，数据库可连接

### Task 0.3: 设计系统初始化

- **目标**: 建立主题系统、基础组件库
- **涉及文件**: `mini-program/theme/`, `mini-program/components/ui/`
- **实现步骤**:
  1. 创建色彩变量系统 (含四季切换)
  2. 创建字体系统
  3. 创建间距系统
  4. 实现 Button / Card / Tag / Input 基础组件
  5. 实现页面容器组件
  6. 实现底部导航组件
- **测试要求**: 组件在模拟器各尺寸正常渲染
- **完成标准**: 5个基础组件可用

## 3. Phase 1: 核心基础设施 (Week 3-6)

### Task 1.1: 用户系统

- **目标**: 微信授权登录 + JWT认证 + 用户数据模型

- **实现步骤**:
  1. 实现微信 `wx.login()` 获取code
  2. 后端实现 code2Session 换取OpenID
  3. 实现JWT签发与验证中间件
  4. 创建User数据模型
  5. 实现用户资料CRUD
  6. 实现Token刷新机制
- **测试要求**: 完整登录流程通过，Token过期刷新正常
- **完成标准**: 用户可登录、查看/编辑个人信息

### Task 1.2: 花园数据模型与API

- **目标**: Flower/Seed/Genome数据模型 + CRUD API
- **涉及文件**: `server/services/flower-service/`, shared types
- **实现步骤**:
  1. 实现Genome Schema
  2. 实现Seed Schema + CRUD
  3. 实现Flower Schema + CRUD
  4. 实现CareLog Schema
  5. 创建MongoDB索引
  6. 编写数据验证管道
- **测试要求**: 所有CRUD端点通过集成测试
- **完成标准**: 可创建/查询/更新/删除花朵和种子

### Task 1.3: 花园主页 — 基础框架

- **目标**: 花园主页布局 + 空状态 + 底部导航
- **涉及文件**: `mini-program/pages/garden/`, `mini-program/components/garden-scene/`
- **实现步骤**:

### Task 1.4: 图片存储与CDN

## 4. Phase 2: MVP核心闭环 (Week 7-12)

### Task 2.1: 种子生成系统 (AI接入)

- **目标**: 关键词→AI种子生成全流程
- **涉及文件**: `server/ai-layer/`, `server/services/breed-service/`, `mini-program/pages/breed/`
- **实现步骤**:
  1. 实现LLM语义解析接口 (关键词→Genome参数)
  2. 实现种子生成Prompt模板
  3. 实现AI图像生成调用 (SD/FLUX API)
  4. 实现图像后处理管线
  5. 实现育种页UI (输入→loading→结果)
  6. 实现种子卡片组件
- **测试要求**: 端到端生成10个不同种子，验证一致性
- **完成标准**: 用户输入关键词 → AI生成独一无二的种子

### Task 2.2: 种植与养成系统

- **目标**: 种子种下 → 多阶段生长 → 照料互动
- **涉及文件**: `mini-program/pages/garden/`, `server/services/flower-service/growth.engine.ts`
- **实现步骤**:
  1. 实现种植流程 (选种子 → 种入花盆)
  2. 实现生长阶段管理 (种子→发芽→生长→花苞→盛花→结果→休眠)
  3. 实现照料交互 (浇水/施肥/光照/修剪)
  4. 实现照料反馈动画
  5. 实现生长状态UI更新
  6. 实现生长阶段过渡动画
- **测试要求**: 完整生命周期模拟测试 (可通过加速模式)
- **完成标准**: 花从种子到盛开完整生命周期可达

### Task 2.3: 花朵渲染组件

- **目标**: 真实感花朵分层精灵渲染
- **涉及文件**: `mini-program/components/flower-viewer/`
- **实现步骤**:
  1. 实现分层精灵系统 (底层→顶层)
  2. 实现阶段间过渡动画 (透明度+缩放混合)
  3. 实现呼吸动画
  4. 实现触摸交互 (拖动旋转/捏合缩放)
  5. 实现光照模拟层
  6. 性能优化 (Canvas离屏渲染/帧率控制)
- **测试要求**: 30fps稳定渲染，各阶段过渡流畅
- **完成标准**: 花朵真实感渲染 + 流畅交互

### Task 2.4: 每日种子系统

- **目标**: 每日随机种子掉落 + 领取
- **涉及文件**: `mini-program/pages/breed/` daily模块, `server/services/breed-service/`
- **实现步骤**:
  1. 实现每日种子生成调度
  2. 实现稀有度随机算法
  3. 实现领取UI + 状态管理
  4. 实现连续签到奖励
  5. 实现分享解锁额外槽位
- **测试要求**: 模拟连续7天签到，验证奖励递增
- **完成标准**: 用户每日可领取随机种子

### Task 2.5: 基础对话系统

- **目标**: AI花朵文字对话 (MVP版)
- **涉及文件**: `server/services/chat-service/`, `mini-program/pages/chat/`
- **实现步骤**:
  1. 实现人格引擎 (DNA→基础人格)
  2. 实现LLM对话接口 (带人格Prompt)
  3. 实现对话记忆管理 (最近N轮)
  4. 实现对话页UI

- **测试要求**: 多轮对话连贯性测试
- **完成标准**: 用户可与花进行上下文连贯的对话

### Task 2.6: MVP集成测试与修复

## 7. 风险管理

### 7.1 技术风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| AI图像生成不稳定 | 中 | 高 | 多模型备选 + 降级策略 + 缓存复用 |
| 微信小程序审核不通过 | 中 | 高 | 提前自检 + 准备备审材料 + 渐进提交 |
| AI对话内容安全风险 | 低 | 极高 | 多层内容审核 + 敏感词实时拦截 |
| 性能不达标 (低端机) | 中 | 中 | 降级渲染 + 性能预算 + 真机测试矩阵 |
| AI API成本超预期 | 中 | 中 | 缓存策略 + 限流 + 成本监控告警 |
| 花朵图像一致性差 | 高 | 中 | 精细Prompt工程 + 多阶段校验 + 人工审核队列 |

### 7.2 项目风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 需求蔓延 | 高 | 中 | 严格P0/P1/P2优先级 + 每两周需求评审 |
| 微信API变更 | 低 | 高 | 关注官方公告 + 兼容性测试 |
| 季节系统错过节气 | 中 | 低 | 节气可手动模拟 + 测试模式加速 |

### 7.3 关键依赖

```
外部依赖清单:
├── 微信小程序审核 (不可控, 需预留时间)
├── AI模型API稳定性 (Stable Diffusion / LLM)
├── 腾讯云服务可用性 (COS / 云开发)
├── 微信API兼容性
└── 第三方NLP审核服务
```
